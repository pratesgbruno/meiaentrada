# meiaentrada-org.com

O **webody.com.br** é um projeto inovador que visa proporcionar uma experiência personalizada para seus usuários, permitindo que eles naveguem por conteúdos exclusivos e adaptados às suas preferências. A plataforma oferece diversas funcionalidades, como vídeos, músicas, e interações digitais, tudo integrado com inteligência artificial para melhorar a experiência do usuário.

## 🚀 Começando

Essas instruções ajudarão você a obter uma cópia do projeto funcionando em sua máquina local para fins de desenvolvimento e teste.

Consulte **[Implantação](#-implantação)** para saber como implantar o projeto.

### 📋 Pré-requisitos

Para rodar este projeto localmente, você precisará ter instalado:

- Node.js (versão 14 ou superior)
- Git
- npm ou yarn como gerenciador de pacotes

### 🔧 Instalação

Siga os passos abaixo para configurar o ambiente de desenvolvimento:

1. Clone o repositório do projeto:
   ```bash
   git clone git@github.com:pratesgbruno/webody.com.br.git
   ```
   
2. Navegue até o diretório do projeto:
   ```bash
   cd webody.com.br
   ```
   
3. Instale as dependências do projeto:
   ```bash
   npm install
   ```
   ou
   ```bash
   yarn install
   ```

4. Inicie o servidor de desenvolvimento:
   ```bash
   npm run dev
   ```
   ou
   ```bash
   yarn dev
   ```

Isso deve abrir o projeto em `http://localhost:3000` em seu navegador padrão.

## ⚙️ Executando os testes

Para garantir que tudo esteja funcionando corretamente, você pode executar os testes automatizados.

### 🔩 Analise os testes de ponta a ponta

Os testes de ponta a ponta verificam a funcionalidade completa da aplicação, simulando o comportamento do usuário final.

```bash
npm run test:e2e
```

### ⌨️ Testes de estilo de codificação

Os testes de estilo de codificação garantem que o código segue as convenções estabelecidas para o projeto.

```bash
npm run lint
```

## 📦 Implantação

Para implantar o projeto em produção, siga os passos abaixo:

1. Faça o build do projeto:
   ```bash
   npm run build
   ```
   ou
   ```bash
   yarn build
   ```

2. Suba os arquivos para o servidor de produção.

3. Configure o servidor para servir os arquivos estáticos gerados no diretório `dist`.
